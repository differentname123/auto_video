现在我有一个需求如下，需要一个完整的提示词来让大模型来实现这个需求，请帮我把下面的需求转换为一个完整的提示词：
1.目标就是为已有的视频选择出合适的封面，我输入数据除了候选的图片外还有视频的信息，是一个dict，示例如下,title是视频标题，cover_text是待会会显示在封面上的文字，也就是说最终的封面是选择出来的图片加上cover_text，video_abstract是视频的简介， solution_idea是视频的思路：
  {
    "title": "BLG这精神病阵容也能赢？BFX信了邪，被G2零封真不冤！",
    "cover_text": "战术幻觉 致命反噬",
    "video_abstract": "复盘BLG险胜背后的蝴蝶效应：一场离谱BP如何葬送BFX的前程。",
    "solution_idea": "本方案通过‘赛前狂言-BP陷阱-绝境救主-战术反噬’的逻辑链条，将一场单一比赛升华为电竞心理博弈。旁白重塑去除了冗余的解说腔调，强化了‘错觉’与‘惩罚’的因果关系。视觉上通过高信息密度的花字锚定关键转折点。"
  }
2.每个视频信息都会有这四个字段，而且每次输入的视频信息肯定是不一样的，所有整个提示词就应该是通用的能够应对不同的视频信息输入。

3.现在我附件会提供多张图片，希望能够为每一张图片进行打分，0到10分，作用是筛选出最适合作为封面的图片.
4.封面文字的区域是固定的，它会直接贴在整张图片的最顶部偏下一点点，所以这个区域是不能有重要信息的，如果这个区域有重要信息的话，那么这个图片就不适合做封面了。
5.有下列情况的得分一定非常的低：
 1.封面文字的区域有重要的信息，因为如果该位置有重要信息，后续把封面文字加上之后就会遮挡重要信息，那么这个图片就不适合做封面了。
 2.封面上已有很大的文字也不行，那就说明这个图片已经是被处理过的
 3.图片是模糊的或者质量很差的，这样的图片也不适合做封面。
6.最后输出的内容必须是存粹的json格式，因为我后续要直接用代码处理，示例的格式如下：
[
{
"image_name": "image1.jpg",
"score": 8,
"reason":"得分的原因"
},
...
]
提示词其它要求如下：
1. 为模型设计，而非为文采
    核心思想: 指令的清晰度和结构性远比语言的华丽更重要。模型是逻辑驱动的，需要的是无歧义、结构化的输入。
    行动指令: 在设计提示词时，优先提供明确的背景(Context)、角色(Role)、限制(Constraints)和输出格式(Format)，以减少模型的猜测空间，确保它能精确理解任务。
2. 以终为始，过程引导
    核心思想: 先清晰定义最终想要获得的理想输出，然后反向推导并提供所有必要的前提条件、信息和思考步骤。
    行动指令: 避免直接命令“给我结果”，而是通过提供上下文和分步指令，引导模型“像这样思考，以达成那个结果”。这本质上是为模型构建一条清晰的“思维链”（Chain of Thought）。
